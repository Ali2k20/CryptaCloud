Abstract

This project introduces an application designed to let users search through their files while ensuring they remain fully encrypted and safe from the cloud provider. Traditional cloud storage requires you to trust the provider, but this application removes that requirement.

The core outcome is that the cloud provider acts essentially as "dumb storage." It only stores files with random, scrambled filenames (like a8f9...enc) and has no way to read the contents or understand what the files are.

Meanwhile, the application on the user's device allows them to see the real filenames and search the file contents to find exactly what they need. This ensures complete privacy without sacrificing the ability to organize and find data easily.

Methodology

A clear separation between trusted local operations and untrusted cloud storage.

  • 1

    User Device (Trusted)

    All operations (encryption, search, decryption) happen here on your device before uploading to the cloud.

  • 2

    Private Search Map

    A private, hidden map stored on your computer that remembers which encrypted file contains which words.

  • 3

    Dumb Storage (Untrusted)

    The cloud storage layer. It stores all our encrypted files, but it has no keys, no logic, and cannot read anything.

Results

CryptaCloud was tested to ensure that the privacy mechanism works as intended.

  • Encrypted Upload: Files are stored with random names (e.g., uuid-123.enc).
  • Private Search: You can find files by content without the cloud knowing the keywords.
  • Tamper Detection: If the cloud file is changed, the app rejects it.

Conclusion
